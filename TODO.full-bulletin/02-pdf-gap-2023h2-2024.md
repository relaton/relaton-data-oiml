# 02 — Process the 5-issue 2023H2 → 2024 PDF gap

## Problem

The dataset has a gap between the docx end (mid-2023) and HTML start
(2025-02). Specifically these issues are missing:

- 2023-Q4 (October 2023) — if not in docx
- 2024-Q1 (January 2024)
- 2024-Q2 (April 2024)
- 2024-Q3 (July 2024)
- 2024-Q4 (October 2024)

All 5 exist as born-digital PDFs on oiml.org. None are in the dataset.

## Solution

Write `backfill/process_pdf_gap.rb` — a one-off script that:

1. Downloads each of the 5 PDFs to `backfill/cache/pdfs/`
2. Extracts text via `pdftotext` (poppler) — born-digital PDFs have a
   clean text layer
3. Parses the SOMMAIRE / Contents page (typically page 2-3) to extract:
   - article title
   - author / affiliation
   - start page
4. Emits the 4-tier bulletin hierarchy for each issue:
   - `data/bulletin_2023.yaml` / `bulletin_2024.yaml` (volumes)
   - `data/bulletin_2023-04.yaml`, `bulletin_2024-01.yaml`, etc. (issues)
   - `data/bulletin_<slug>-<seq>.yaml` (articles)
5. Patches `data/bulletin.yaml` to add 2023 / 2024 to hasPart if missing

For 2024-07 and 2024-10 specifically, the HTML editions exist (once
TODO 01 lands) — prefer HTML extraction there, fall back to PDF only
if the HTML fetch fails.

## Acceptance

- The 5 issues are present in `data/` with their articles
- Each new article has: title, author (where the SOMMAIRE lists one),
  start page, source URL
- `bundle exec ruby check_data.rb` passes for all new files
- Index updated; `data/bulletin.yaml` lists the 2023 and 2024 volumes
